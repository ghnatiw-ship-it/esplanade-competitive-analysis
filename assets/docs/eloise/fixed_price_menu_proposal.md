# Eloise — Peak-Time Experience Proposal

> Date: April 13, 2026
> Objective: protect Thu-Sat 6:30-8:00 p.m. inventory for guests who land the target average cheque
> Mechanism: OpenTable `Experience` that repackages existing Eloise dishes into a required premium reservation type

## Recommendation

Do not frame this as a minimum spend.

Use an OpenTable `Experience` that is only offered on the highest-demand dinner times and is described as a curated Eloise dinner. That is how stronger restaurants on OpenTable are handling the same commercial goal.

Recommended first test:

- Name: `Taste Eloise Signature`
- Days: Thursday to Saturday
- Times: `6:30 p.m.`, `7:00 p.m.`, `7:30 p.m.`
- Party size: `2-6`
- Rule: whole-table participation
- Format: 4 courses using existing menu items
- Price: `CA$110` per guest
- Optional wine pairing: `+CA$45`

## Why This Format Works

- It protects the best reservation times without discounting anything.
- It feels curated and premium instead of punitive.
- It uses dishes already on the current menu.
- It follows real OpenTable precedents rather than inventing a new behavior.
- It can be limited to only part of peak inventory at launch.

## Target Cheque Math

### Scenario A: Target average cheque `CA$95`

- Base menu price: `CA$95`
- Alcohol upside: incremental
- Best if you want lower booking friction

### Scenario B: Target average cheque `CA$110`

- Base menu price: `CA$110`
- Pairing optional at `+CA$45`
- Best balance of conversion and revenue

### Scenario C: Target average cheque `CA$125`

- Base menu price: `CA$110`
- Pairing uptake or supplement pushes final cheque above `CA$125`
- Better than pricing the base menu too high on day one

Recommended commercial logic:

- Use the menu itself to guarantee `CA$110`
- Use pairings and supplements to push the realized cheque to `CA$125+`

## Inventory Setup

Do not convert every prime-time table on day one.

Recommended launch mix:

- Hold `35-50%` of Thu-Sat `6:30-7:30` inventory for the Experience
- Keep the balance as standard dinner while testing conversion

Recommended release rules:

- Thursday unsold experience inventory releases back to standard dinner `72 hours out`
- Friday unsold inventory releases `48 hours out`
- Saturday unsold inventory releases `24 hours out`

This protects revenue without creating artificial empty seats.
